想象大海中的海浪

Picture a wave in the ocean.

可看 可量 高度可见

You can see it, measure it, its height,

阳光洒在波浪上的斑驳光点

the way the sunlight refracts when it passes through,

永远在海洋中

and it’s there,

看得见 摸得着

and you can see it, you know what it is.

就是海浪

lt’s a wave.

浪花打在海岸上

And then it crashes on the shore,

瞬间消失 但水还在

and it’s gone.But the water is still there.

海浪只是

The wave was just a–

海水短暂存在的一种方式

a different way for the water to be for a little while.

这是佛教徒对死亡的见解

That’s one conception of death for a Buddhist.

海浪回归大海

The wave returns to the ocean,

尘归尘 土归土

where it came from,

从来处来 往去处去

and where it’s supposed to be.

基于Windows的求生之路2（8人战役）服务器搭建

发表于 2022-02-15 分类于 游戏
本文字数： 1k 阅读时长 ≈ 1 分钟 
更新日志：
 2022年2月15日（第三部分完成，其余部分不可用。）
1.购买云服务器
2.通过供应商提供的接口远程连接至云服务器
3.在云服务器部署SteamCMD和求生之路2服务端
3.1在云服务器上下载SteamCMD

（1）在云服务器中新建一个文件夹用于存放SteamCMD和steam服务端组件

　　例如在C盘中新建一个文件夹，名为：steamcmd。

（2）在云服务器上下载SteamCMD（Windows）并解压至步骤（1）的新建文件夹中

　　官方下载链接：https://steamcdn-a.akamaihd.net/client/installer/steamcmd.zip

3.2在云服务器上使用脚本部署steam服务端和求生之路2服务端

（1）在SteamCMD所在文件夹下（步骤3.1中新建的文件夹）新建一个.bat脚本

可先新建一个.txt(文本文档)，例如：新建一个名为“start”的文本文档。

（2）打开新建的脚本，输入脚本内容。

　　脚本内容如下（复制粘贴即可）：

start steamcmd.exe +login anonymous +force_install_dir 14d2_sv +app_update 222860 validate +quit

　　输入内容后，关闭文本文档，将其后缀修改为.bat。例如，将start.txt重命名为start.bat。

（3）双击运行脚本，等待steam服务端和求生之路2服务端下载完成。若下载成功，文件夹中将出现名为l4d2_sv的子文件夹，其中为求生之路2服务端的组件。

4.进行求生之路2服务端初始化配置

\l4d2_sv\left4dead2\cfg\server.cfg

　　打开l4d2_sv文件夹中的left4dead2文件夹，再打开cfg文件夹，该文件夹中应有一个名为server.cfg的配置文件（若不存在可以自己创建一个）。用记事本打开该配置文件，输入以下内容（复制粘贴即可）：

sv_tags "hidden"

//player number
l4d_maxplayers 8
sv_maxplayers 8
sv_visiblenmaxplayers 8
sv__removehumanlimit "1"
 
//game mod
sm_cvar mp_gamemode "coop"

 // lobby
sv_allow_lobby_connect_only 0
5.插件安装（选）

Markdown语言学习

发表于 2022-02-15 更新于 2022-02-20 分类于 学习笔记
本文字数： 1.2k 阅读时长 ≈ 1 分钟

1. Markdown语言概述

1.1 Markdown简介

图1-1 markdown图标

　　Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档。Markdown 语言在 2004 由约翰·格鲁伯（英语：John Gruber）创建。Markdown 编写的文档可以导出 HTML 、Word、图像、PDF、Epub 等多种格式的文档。Markdown 编写的文档后缀为 .md, .markdown。

1.2 Markdown应用

图1-2 github

　　Markdown 能被使用来撰写电子书，如：Gitbook。当前许多网站都广泛使用 Markdown 来撰写帮助文档或是用于论坛上发表消息。例如：GitHub、简书、reddit、Diaspora、Stack Exchange、OpenStreetMap 、SourceForge等。

2. Markdown标题

2.1 使用#号标记

　　使用 # 号可表示 1-6 级标题，一级标题对应一个 # 号，二级标题对应两个 # 号，以此类推。注意：#号后需空一格再进行标题的书写。

例：

 # 一级标题
 ## 二级标题

2.2 使用=号和-号标记一级标题和二级标题

例：

我展示的是一级标题
=================

我展示的是二级标题
-----------------

　　本方法的局限性是只能标记两级标题。

3. Markdown段落格式

3.1 换行

　　段落的换行是使用两个以上空格加上回车或使用一个空行来换行。

3.2 字体

　　Markdown 可以使用以下几种字体：

　　①斜体：在文本两端分别添加一个*号或一个_（下划线）可将其中间的文本字体设置为斜体。

例：

*斜体文本*
_斜体文本_

　　显示效果均为：斜体文本

　　②粗体：在文本两端分别添加两个*号或两个_（下划线）可将其中间的文本字体设置为粗体。

例：

**粗体文本**
__粗体文本__

显示效果均为：粗体文本

　③粗斜体：在文本两端分别添加三个*号或三个_（下划线）可将其中间的文本字体设置为粗斜体。

例：

***粗斜体文本***
___粗斜体文本___

显示效果均为：粗斜体文本

3.3 分隔线

　　可以在一行中用三个以上的星号、减号、底线来建立一个分隔线，行内不能有其他东西。你也可以在星号或是减号中间插入空格。下面每种写法都可以建立分隔线：

***

* * *

*****

- - -

----------

其效果均为：

3.4 删除线

　　如果有文本需要添加删除线，只需要在文字的两端分别加上两个波浪线 ~~ 即可，

实例如下：

~~删除线~~

　　显示效果为：删除线

3.5 下划线

　　下划线可以通过 HTML 的 <u> 标签来实现：

<u>带下划线文本</u>

　　显示效果为：带下划线文本

3.6 脚注

　　脚注是对文本的补充说明。

　　Markdown 脚注的格式如下:

[^脚注名]
[^脚注名]: 具体的注释

　　显示效果为：实例文本^脚注名
